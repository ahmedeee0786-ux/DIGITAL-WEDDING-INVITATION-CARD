# 💍 Universal Digital Wedding Invitation Card

A premium, professional, and fully customizable digital wedding invitation system. Designed for ease of use, high-speed sharing, and a stunning guest experience.

![Wedding Card Preview](https://img.shields.io/badge/Status-Production--Ready-brightgreen)
![License](https://img.shields.io/badge/License-MIT-blue)

## ✨ Key Features

- **🎨 10 Premium Themes**: Choose from Traditional Teal, Royal Blue, Luxury Black & Gold, Vintage Cream, and more.
- **🎵 Personalized Music**: Support for 4 background tracks. The host can choose a specific song for each guest link.
- **✉️ Personal Guest Welcome**: Every link can be personalized with the guest's name (e.g., "Warmest Welcome, Ali!").
- **📱 WhatsApp RSVP System**: Guests can confirm their attendance with one click, sending an automated message to the host.
- **📍 Google Maps Integration**: Direct navigation link to the wedding hall/marquee.
- **⏳ Live Countdown**: Real-time countdown to the big day.
- **🚀 High-Compression Links**: Unique guest links are compressed to ensure they remain "clickable" on all messaging apps like WhatsApp.

## 🚀 How to Use

### 🤵 For the Host (Owner)
1. **Open the site**: Access the main URL.
2. **Customize**: Click the **"EDIT YOUR CARD"** button at the bottom right.
3. **Set Details**: Enter the Groom & Bride names, event dates, and locations.
4. **Choose Music & Theme**: Pick a color palette and a background song that fits your guest.
5. **Personalize**: Enter the Guest's Name and your WhatsApp number for RSVP.
6. **Share**: Click **"Share on WhatsApp"** to send the invitation directly.

### 👤 For the Guest
1. **Open the Link**: Tap the unique link received.
2. **Experience**: Click **"TAP TO OPEN"** to hear the music and see the card.
3. **RSVP**: Use the buttons at the bottom to notify the host of your attendance.
4. **Navigate**: Use the "View on Maps" button to get directions.

## 🛠️ Technical Details

- **Frontend**: Vanilla HTML5, CSS3, JavaScript.
- **Storage**: None! This is a **serverless** project. All data is encoded directly into the URL using Base64 and flat-string serialization.
- **Deployment**: Optimized for GitHub Pages.
